# 리포트아이 v13 — 분석 정확도 개선

## 핵심 원칙 변경
**"PDF 원본이 진실, 로컬파서는 참고만"**

이전(v12)까지는 로컬 정규식 파서가 추출한 데이터로 Claude의 결과를 덮어쓰는 구조였습니다.
그래서 Claude가 PDF를 직접 보고 정확히 분석해도, 정규식이 표 구조를 잘못 읽으면
최종 결과가 망가졌습니다. v13에서 이 구조를 뒤집습니다.

---

## 수정된 파일 3개

### 1. `src/criteria.js` — 프롬프트 재작성
**변경 요지:**
- 프롬프트 맨 앞에 "PDF가 진실, 로컬파서는 보조" 원칙 명시
- "로컬파서 결과를 grades에 그대로 써라" 지시 → "PDF에서 직접 읽어라"로 변경
- 로컬파서 데이터는 참고용 힌트로만 (앞 10개만 요약)
- 원문 텍스트도 30000자로 축소 (PDF 원본이 있으므로)

### 2. `api/analyze.js` — 모델 업그레이드 + 덮어쓰기 약화
**변경 요지:**
- 모델: `claude-sonnet-4-20250514` → `claude-sonnet-4-5` (Sonnet 4.6, 같은 가격에 성능↑)
- max_tokens: 8000 → 16000 (응답 잘림 방지)
- GPT 폴백에서 로컬파서로 grades 덮어쓰기 제거
- 에러 메시지 구체화 (한글파일 안내 추가)

### 3. `src/App.jsx` — run 함수 4곳 수정
**변경 요지:**
- **변경 1**: 로컬 파싱 실패해도 진행 (return으로 끊지 않음)
- **변경 2**: 텍스트 추출 실패 시 빈 객체로 채워서 Claude한테 PDF만 보냄
  → 이미지 기반 PDF/보안 PDF도 분석 가능
- **변경 3**: Claude 결과를 로컬파서로 덮어쓰는 로직 ★제거★ (가장 중요)
- **변경 4**: Claude가 빈 결과를 줬을 때만 로컬파서 폴백 사용

## 적용 방법

### Cursor에서
1. 이 zip 압축 풀기
2. 본인 프로젝트의 다음 3개 파일을 교체:
   - `src/criteria.js`
   - `src/App.jsx`
   - `api/analyze.js`
3. 나머지 파일(parser.js, App.css 등)은 그대로 두기
4. `npm run dev`로 로컬 테스트
5. Git push → Vercel 자동 배포

### 환경변수 확인
Vercel 대시보드 → Settings → Environment Variables:
- `ANTHROPIC_API_KEY` (필수, Sonnet 4.6 호출용)
- `OPENAI_API_KEY` (선택, GPT-4o 폴백용)

## 테스트 체크리스트

배포 후 확인할 것:
- [ ] 텍스트 기반 PDF: 성적이 정확하게 추출되는가
- [ ] 학년/학기가 올바르게 매핑되는가 (1학년 성적이 1학년 칸에 가는가)
- [ ] 진로선택과목(성취도 A/B/C)이 achievementSubjects에 따로 분리되는가
- [ ] 이미지 기반 PDF: "텍스트 추출 실패" 안 뜨고 분석 진행되는가
- [ ] 활동(activities)에서 원문 인용이 정확한가
- [ ] 응답이 중간에 잘리지 않는가 (max_tokens 16000 효과)

## 알려진 한계

1. **로컬 파서(parser.js)는 그대로** — 폴백용으로만 쓰임. 시간 되면 정규식 보강 가능.
2. **이미지 PDF에서 Claude도 못 읽는 경우** — 매우 드물지만 가능. 이 경우 결과 부분 비어있을 수 있음.
3. **분석 시간** — 16000 토큰 출력이라 60~90초 걸릴 수 있음.
